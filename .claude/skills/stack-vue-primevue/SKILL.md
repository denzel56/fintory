# Vue PrimeVue UI

- Prefer PrimeVue components before custom UI.
- Use `DataTable` for transaction lists and rich tabular data.
- Use `Dialog`, `Drawer`, `Toast`, `Menu`, and form components consistently.
- Keep theme, spacing, and visual hierarchy consistent.
- Provide clear empty, loading, and error states.
- Avoid complex template logic inside PrimeVue slots; extract it to computed values or methods.
- Ensure desktop and small-window layouts remain usable.
