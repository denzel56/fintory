# Product Principles

- Optimize for the primary user workflow.
- Prefer the smallest useful feature over a broad feature set.
- Avoid speculative architecture and future-proofing unless the requirement is concrete.
- Make tradeoffs explicit when scope, complexity, or user value conflict.
- Preserve user trust: do not add tracking, hidden network behavior, or surprising data usage without explicit requirements.
