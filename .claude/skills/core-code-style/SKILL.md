# Core Code Style

- Prefer strong, explicit types at module boundaries.
- Do not use `any` unless there is a documented reason.
- Keep functions focused and easy to read.
- Prefer clear names over comments.
- Avoid premature abstractions.
- Reuse existing utilities and patterns before creating new ones.
- Keep imports consistent with the repository style.
- Do not add dead code, commented-out code, secrets, or debug logs.
- Make the smallest correct change that satisfies the task.
